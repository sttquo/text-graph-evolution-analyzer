# Text Graph Evolution Analyzer

Программа анализирует изменения между двумя версиями текста и показывает их как эволюцию графа:
- что сохранено;
- что заменено;
- что удалено;
- что добавлено;
- как меняются синтаксические и семантические связи.

Поддерживаются английские и русские предложения (автоопределение языка).

## Что делает программа

Программа выполняет 7 шагов:
1. Токенизация исходного и отредактированного текста.
2. Выравнивание токенов между версиями.
3. Синтаксический анализ (роли слов и зависимости).
4. Семантический анализ (упрощенные смысловые триплеты).
5. Подсчет статистики изменений.
6. Построение кадров эволюции графа.
7. Визуализация (PNG-изображения).

## Архитектура

Код разделен по блокам:
- `graf.py` — точка входа.
- `textgraph/cli.py` — консольное меню.
- `textgraph/pipeline.py` — общий пайплайн анализа.
- `textgraph/parsing.py` — токенизация и выравнивание.
- `textgraph/analysis.py` — синтаксис, семантика, dependency-ребра.
- `textgraph/evolution.py` — построение последовательности кадров.
- `textgraph/visualization.py` — отрисовка графов и кадров.
- `textgraph/reporting.py` — формирование текстовых отчетов.
- `textgraph/language.py` — определение языка (`ru`/`en`).
- `textgraph/models.py` — dataclass-модели.

## Установка

Требуется Python 3.10+ (рекомендуется 3.12+).

Установка зависимостей для графиков:

```powershell
py -3 -m pip install networkx matplotlib
```

Если зависимости не установлены, текстовые отчеты все равно будут сохранены, но PNG-графики не будут построены.

## Запуск

Из корня проекта:

```powershell
py -3 graf.py
```

В меню:
- `1` — запустить встроенные примеры.
- `2` — ввести свои два текста.
- `3` — выход.

## Где лежат результаты

Для каждого анализа создается отдельная папка:

`results/results_YYYYMMDD_HHMMSS_microseconds/`

Внутри:
- `0_original_texts.txt` — обе версии текста.
- `1_alignment.txt` — выравнивание токенов.
- `2_syntax_analysis.txt` — сравнение синтаксических ролей.
- `2a_dependencies_original.txt` — dependency-ребра исходного текста.
- `2b_dependencies_edited.txt` — dependency-ребра отредактированного текста.
- `3_semantic_analysis.txt` — семантические триплеты и их изменения.
- `4_statistics.txt` — агрегированная статистика.
- `5_evolution_report.txt` — информация по кадрам эволюции.
- `syntax_original.png` — граф зависимостей исходного текста.
- `syntax_edited.png` — граф зависимостей отредактированного текста.
- `frames/` — 6 PNG-кадров эволюции текста.

## Как читать результаты

### 1) Выравнивание (`1_alignment.txt`)
- `СОХРАНЕНО` — слово сопоставилось с тем же словом.
- `ЗАМЕНЕНО` — найдено похожее/синонимичное соответствие.
- `УДАЛЕНО` — слово было в исходном тексте, но не попало в редакцию.
- `ДОБАВЛЕНО` — слово появилось только в редакции.

### 2) Синтаксис (`2_syntax_analysis.txt`)
Показывает роль слова в исходном и новом тексте и сохранилась ли роль.

### 3) Зависимости (`2a/2b_dependencies_*.txt`)
Строка формата:

`head_token -> dependent_token (relation)`

Это связи между словами, которые используются для построения синтаксического графа.

### 4) Семантика (`3_semantic_analysis.txt`)
- Общие триплеты = смысловые инварианты.
- `Удалено` = смысл, который пропал после редактуры.
- `Добавлено` = новый смысл в редакции.

### 5) Статистика (`4_statistics.txt`)
Краткая метрика по всем типам изменений: сохранение, замена, удаление, добавление.

### 6) Эволюция (`5_evolution_report.txt` + `frames/*.png`)
Покадрово показывает переход от исходного текста к отредактированному.

## Ограничения

- Это эвристический анализатор (не полноценный NLP-парсер).
- Качество зависит от словарей ролей и синонимов в `analysis.py`/`parsing.py`.
- Для узкоспециальной лексики лучше расширять словари под предметную область.

## Как расширять

1. Добавлять новые слова и роли в `SYNTAX_ROLES_EN` / `SYNTAX_ROLES_RU` в `textgraph/analysis.py`.
2. Расширять словари синонимов в `textgraph/parsing.py`.
3. Добавлять семантические шаблоны триплетов в `extract_semantic_triples`.
4. Настраивать правила dependency-связей в `build_dependency_edges`.
